# FeatherSpace Demo Runbook

## Objective
Present a stable MVP journey in under 4 minutes with no detours into non-essential controls.

## Pre-Demo Setup
1. Start the client app.
2. Enable Presentation Mode using the top-right toggle.
3. Open the guided flow from the Overview page.

## Live Walkthrough Script
1. Overview
- Line: "This is FeatherSpace's core MVP flow for room discovery, live runtime interaction, and operational visibility."
- Action: Click "Start guided demo".

2. Rooms (Step 1)
- Line: "The launch form is prefilled for our research studio template, including capacity and communication defaults."
- Action: Click "Continue to live room".

3. Live Room (Step 2)
- Line: "This runtime view shows participant context, overlay signals, and realtime connectivity status in one screen."
- Action: Point out participant rail and connection panel.
- Action: Click "Open operations board".

4. Operations
- Line: "Operations reflect current runtime mode and realtime state, so health reporting stays aligned with deployment conditions."
- Action: Point out room sync, signaling relay, and realtime status line.

## Safety Notes
- If realtime backend is unavailable, call out "Demo-safe mode" explicitly.
- Stay on Overview -> Rooms -> Live Room -> Operations for consistency.
- Avoid off-script routes while presenting.

## Q&A Backup
- If asked about builder depth: "The builder now validates constraints and generates live config snapshots; integration with publish APIs is the next milestone."
- If asked about reliability: "Realtime status and reconnection messaging are surfaced in both room runtime and operations."
