# FeatherSpace Documentation Map

Use this folder as the single coordination hub for your capstone implementation.

## Read in This Order

1. `docs/coordination_index.md`
- Canonical architecture and contracts.

2. `docs/platform_details.md`
- Full technical specification and scope details.

3. `docs/agent_prompt_playbook.md`
- Copy-paste prompts for each specialized agent.

4. `docs/delivery_protocol.md`
- Team workflow, ownership rules, and definition of done.

## Quick Start for Team Coordination

1. Assign module ownership from `docs/delivery_protocol.md`.
2. Start every task using the mandatory task template.
3. Use role prompts from `docs/agent_prompt_playbook.md`.
4. Keep all changes aligned to contracts in `docs/coordination_index.md`.
5. Finish each task with a handoff summary.

## Why This Structure Works

- Reduces agent-to-agent drift.
- Keeps architecture decisions explicit.
- Prevents accidental server-heavy implementation.
- Speeds up onboarding for new teammates.
